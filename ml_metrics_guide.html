üéØ 4. Precision

What is it?

"When my model says YES, how often is it actually correct?"

Precision = TP / (TP + FP)

Out of all the things I predicted as positive, how many were truly positive?

üìå Spam Email Example:

Your spam detector flagged 60 emails as spam:

  • 50 were actually spam (TP = 50) ‚úÖ
  • 10 were normal emails (FP = 10) ‚ùå (False alarms!)

Precision = 50 / (50 + 10) = 50/60 = 83.3%

This means: When the model says "spam", it's right 83.3% of the time.

ü§î When is Precision important?

High Precision Matters When:

  • Email spam filters: Don't want important emails going to spam (false positives are bad!)
  • Product recommendations: Don't want to show irrelevant products
  • Fraud detection: Don't want to block legitimate transactions

Rule of thumb: Use precision when false alarms are costly.

üéØ Real-world trade-off:

Scenario: YouTube's copyright detection system

  • Low Precision: Flags lots of videos as copyright violations, including many legitimate ones (creators angry!)
  • High Precision: Only flags videos it's very sure about (fewer false alarms, happier creators)

üîç 5. Recall (Sensitivity)

What is it?

"Out of all the actual positives, how many did I catch?"

Recall = TP / (TP + FN)

Out of all the things that ARE positive, how many did I find?

üìå Disease Detection Example:

There are 100 people with a disease:

  • Your model detected 80 of them (TP = 80) ‚úÖ
  • Missed 20 sick people (FN = 20) ‚ùå (Dangerous!)

Recall = 80 / (80 + 20) = 80/100 = 80%

This means: The model caught 80% of sick people, but missed 20%.

ü§î When is Recall important?

High Recall Matters When:

  • Cancer screening: Can't afford to miss sick patients (false negatives are deadly!)
  • Fraud detection: Need to catch as many fraud cases as possible
  • Airport security: Can't let threats slip through
  • Missing person search: Need to check every possible lead

Rule of thumb: Use recall when missing positives is dangerous.

üéØ Real-world trade-off:

Scenario: Airport security scanning

  • Low Recall: Misses some threats (very dangerous!)
  • High Recall: Flags almost everything suspicious (many false alarms, but safer)

They choose high recall - better safe than sorry!

üí° Which should you prioritize?

  • Prioritize Precision when: False alarms are expensive (spam filters, recommendations)
  • Prioritize Recall when: Missing positives is dangerous (medical diagnosis, security)
  • Balance both when: Both types of errors matter equally

üéØ Quick Decision Guide:

Ask yourself: "What's worse - a false alarm or missing something important?"

  • False alarm worse? ‚Üí Optimize for Precision
  • Missing something worse? ‚Üí Optimize for Recall

üéØ 6. F1 Score

What is it?

The "best of both worlds" metric - combines Precision and Recall into one number.

F1 Score = 2 √ó (Precision √ó Recall) / (Precision + Recall)

It's the harmonic mean of Precision and Recall

üìå Why not just average them?

F1 Score punishes extreme imbalances:

Precision Recall Simple Average F1 Score
100% 100% 100% 100%
90% 90% 90% 90%
100% 10% 55% (misleading!) 18% (realistic!)
10% 100% 55% (misleading!) 18% (realistic!)

ü§î When should you use F1 Score?

  • When you need both Precision and Recall to be good
  • When you want one number to judge overall performance
  • When your dataset is imbalanced (like 95% negative, 5% positive)
  • When you can't decide which is more important - Precision or Recall

üéØ Real Example - Medical Test:

Testing for a rare disease (1% of population has it):

Model A - "The Conservative"
  • Precision: 95% (when it says sick, usually right)
  • Recall: 30% (but misses 70% of sick people!) ‚ö†Ô∏è
  • F1 Score: 46% (reveals it's actually poor!)
Model B - "The Balanced"
  • Precision: 85% (some false alarms)
  • Recall: 85% (catches most sick people)
  • F1 Score: 85% (much better overall!)

Model B is clearly superior - F1 Score revealed this!

üí° Pro Tip:

F1 Score ranges from 0 to 1 (or 0% to 100%):

  • F1 = 1.0 (100%) ‚Üí Perfect precision AND recall
  • F1 ‚â• 0.8 (80%) ‚Üí Good balance
  • F1 < 0.5 (50%) ‚Üí Poor, needs improvement

üéØ Analysis:

  • ‚úÖ High accuracy (94%) - but misleading due to 900 healthy vs 100 sick
  • ‚ö†Ô∏è Medium precision (65%) - many false alarms (not ideal, but acceptable)
  • ‚ùå Good recall (85%) - but missing 15 cancer patients is dangerous!
  • üìä F1 Score (74%) - shows room for improvement

üí° Recommendation:

Increase recall to 95%+ even if precision drops slightly. In medical diagnosis, missing sick patients is more dangerous than false alarms!

Question 1:

You build a spam email detector:

  • Accuracy: 98%
  • But 40% of real spam gets through (low recall)
  • When it flags spam, it's right 95% of the time (high precision)

What's the main problem?

Answer:

Low Recall is the problem!

  • High accuracy (98%) is misleading - probably because most emails aren't spam
  • High precision (95%) means few false alarms ‚úÖ
  • But low recall (60%) means 40% of spam is getting through! ‚ùå

Solution: Make the spam filter more aggressive to catch more spam, even if it means a few more false positives.

Question 2:

Two models for detecting fraudulent transactions:

  • Model A: Precision = 99%, Recall = 50%, F1 = 66%
  • Model B: Precision = 80%, Recall = 90%, F1 = 85%

Which model should you choose and why?

Answer:

Model B is better!

  • Model A: Very precise but misses 50% of fraud (dangerous!)
  • Model B: Catches 90% of fraud with acceptable precision

Why: In fraud detection, missing fraud (low recall) is worse than a few false alarms. Model B's higher F1 score (85% vs 66%) confirms it's better balanced.

The F1 Score correctly identified Model B as superior!

Question 3:

Your model has:

  • Petal Width importance: 60%
  • All other features: 40% combined

Petal Width dominates the decision!

Action: Consider if this makes sense for your problem. Is petal width truly the most important? Or is the model overfitting to this feature?

üéØ Always Use Multiple Metrics Together:

üîç Quick Decision Flowchart:

  1. Is your dataset balanced? ‚Üí Use Accuracy
  2. Is your dataset imbalanced? ‚Üí Use F1 Score
  3. Are false alarms costly? ‚Üí Optimize Precision
  4. Is missing positives dangerous? ‚Üí Optimize Recall
  5. Want to understand errors? ‚Üí Check Confusion Matrix
  6. Want to improve model? ‚Üí Analyze Feature Importance

‚ö†Ô∏è Common Mistakes to Avoid:

  • ‚ùå Using only accuracy on imbalanced datasets
  • ‚ùå Ignoring the confusion matrix
  • ‚ùå Not considering the cost of different types of errors
  • ‚ùå Optimizing for precision when recall matters more (or vice versa)
  • ‚ùå Not checking if important features make domain sense
